# SFD Tool 中文用户指南

> 本文为 SFD Tool 的中文用户手册，主要面向日常刷机 / 备份 / 维护设备的用户。
>
> **警告：错误的刷写与高级操作有极高风险，可能导致设备变砖或数据永久丢失。请务必在充分理解功能含义的前提下操作，并提前做好重要分区备份。**

---

## 1. 工具简介

SFD Tool 是一款针对 **Spreadtrum / UNISOC 平台设备** 的刷机与维护工具，
提供图形界面（GUI）和命令行（CLI）两种使用方式，支持：

- 读取 / 备份 / 还原设备分区；
- 刷写 PAC 固件包；
- 按地址读写、执行代码等手动低级操作；
- 一些高级维护功能（如 AVB 相关操作、调试辅助等）。

本工具支持：

- Windows 10/11 x64/x86；
- Linux x86_64（常见 Debian/Ubuntu、部分 RPM 系发行版）；
- macOS（近期版本）；
- Termux / Android（仅 CLI 模式，通过 `--no-gui`）。

---

## 2. 安装与启动

### 2.1 获取程序

推荐通过项目的 GitHub Releases 页面获取预编译包：

- Windows：下载打包好的 x64 可执行文件（LibUSB / SPRD 驱动版本）；
- Linux：下载 `.deb` / `.rpm` 软件包，并使用系统包管理器安装；
- macOS：下载 `.dmg` 并将应用拖拽至「应用程序」；
- Termux：从源码构建，或使用适配脚本（仅 CLI）。

源码构建方式请参考：

- [docs/cmake.md](docs/cmake.md)（中文 CMake 指南）；
- [docs/cmake_EN.md](docs/cmake_EN.md)（英文说明）。

### 2.2 图形界面启动

安装或构建完成后，可以通过以下方式启动 GUI：

- 在桌面环境的菜单中找到 SFD Tool 图标并点击；
- 或在终端中执行：

```bash
sfd_tool
```

Windows 下可直接双击可执行文件启动。

### 2.3 以命令行模式启动

若只希望使用命令行模式（无图形界面），可以使用参数 `--no-gui`：

```bash
sfd_tool --no-gui --help
```

这会在终端中运行工具，并打印所有可用命令和参数。

> 命令行模式适合在服务器、Termux 或只有串行/终端环境的场景使用。

## 3. 图形界面概览

SFD Tool 的 GUI 使用 GTK3 实现，主要由多个标签页（Tab）组成：

- **连接**（Connect）
- **分区操作**（Partition Operation）
- **PAC 刷写**（PAC Flash）
- **手动操作**（Manual）
- **高级操作**（Advanced Operation）
- **高级设置**（Advanced Settings）
- **调试设置**（Debug Options）
- **日志**（Log）
- **关于**（About）

下文按标签页分别介绍主要功能及常见操作流程。

---

## 4. 连接页（Connect）

连接页是整个工具的起点，用于与设备建立通信。

典型步骤：

1. 将设备置于合适的刷机模式（如开机进入 bootrom / factory 模式等）；
2. 通过 USB 线缆连接设备与电脑；
3. 在连接页中选择合适的端口或设备（如 LibUSB 设备列表中的目标设备）；
4. 点击「连接」按钮。

连接成功后，界面会显示：

- 设备识别信息；
- 当前使用的传输方式（LibUSB / SPRD 驱动等）。

> 如果连接失败，请首先检查驱动是否安装正确、线缆质量以及系统权限（Linux 下通常需要 udev 规则或 root 权限）。

---

## 5. 分区操作页（Partition Operation）

分区操作页用于对单个或多个分区进行备份与还原，是使用频率最高的页面之一。

### 5.1 分区列表

在成功连接设备并刷新分区表后，会显示一个分区列表：

- 分区名称；
- 大小；
- 起始地址；
- 所属槽位（如 A/B）；
- 是否为关键分区等。

通常需先点击「刷新分区表」按钮，从设备获取最新分区信息。

### 5.2 备份单个分区

1. 在分区列表中选中目标分区；
2. 选择保存路径（如使用文件选择对话框）；
3. 点击「备份」或类似按钮开始读取；
4. 进度条及日志会显示读取进度和速率。

> SFD Tool 通过统一的 `PartitionReadService` 和块大小配置，
> 以 **块大小感知（block-size-aware）** 的方式进行读取，避免常见的对齐问题。

### 5.3 从镜像还原单个分区

1. 在分区列表中选中目标分区；
2. 选择待写入的镜像文件；
3. 确认提示（特别是关键分区会有较强警告）；
4. 开始刷写并等待完成。

### 5.4 从文件夹批量还原分区（重点新功能）

当你已经有一个目录，包含多个分区的镜像文件时，可以使用
**「从文件夹还原」** 功能批量还原。

典型流程：

1. 在分区操作页，确保已经成功加载分区表；
2. 点击「从文件夹还原」（Restore From Folder）按钮；
3. 在弹出的文件夹选择对话框中，选择包含分区镜像的目录；
4. 工具会扫描该目录，按规则匹配可识别的分区镜像，例如：
   - `system.img` 对应 `system` 分区；
   - `userdata.img` 对应 `userdata` 分区；
   - 某些设备可能使用类似 `partition_name.bin` 等命名；
5. 将匹配结果以列表形式展示，包含：
   - 将要刷写的分区名称；
   - 对应的镜像文件路径；
   - 大小信息；
6. 你可以在对话框中勾选/取消勾选某些分区；
7. 确认后，工具会按选择顺序依次刷写。

注意事项：

- 未匹配到的文件会被忽略；
- 若分区表中不存在对应分区名，则镜像不会被刷写；
- 某些平台暂不支持目录扫描（如部分 Windows 平台会退回到简化逻辑），
  具体以日志说明为准。

> 强烈建议在批量还原前，对即将刷写的分区做好完整备份，避免
> 镜像错误导致难以恢复的损坏。

### 5.5 块大小模式简介

分区读取与备份过程中，工具使用 **块大小模式（Block Size Mode）**
控制实际读写步长：

- 默认模式下会优先使用握手阶段得到的「协议推荐块大小」，
  若未获取到则退回协议层默认值；
- 手动模式下可以使用固定块大小（在高级设置中配置），
  适合某些设备上更稳定或兼容性更好的值。

一般情况下无需手动修改，只有在遇到特殊设备或
需要测试性能/兼容性时才需要调整。

## 6. PAC 刷写页（PAC Flash）

PAC 刷写页用于加载并刷写官方或第三方 PAC 固件包。

典型流程：

1. 点击「选择 PAC 文件」，选择 `.pac` 固件；
2. 工具解析 PAC，展示其中的分区条目和元信息；
3. 勾选/取消勾选需要刷写的分区；
4. 确认后开始刷写，进度条和日志窗口会提示当前阶段；
5. 刷写完成后根据提示执行重启或拔线。

建议：

- 首次刷写前最好通过分区操作页备份关键分区；
- 避免反复交叉刷写不同版本的 boot / vbmeta 等分区，
  以防引导链出现难以恢复的问题。

---

## 7. 手动操作页（Manual）

手动操作页面向熟悉底层布局的高级用户，常见功能包括：

- 按地址读取指定长度数据到文件；
- 将文件写入指定地址；
- 在指定地址执行代码（`exec_addr` / `exec` 等）。

这些操作不会进行太多安全检查：

- 一旦填写了错误的地址或长度，可能破坏关键区域；
- 在未知代码上执行 `exec` 极易导致系统崩溃或设备变砖。

> 普通用户不建议使用该标签页，除非你非常明确地知道
> 自己正在做什么，且已有完整备份和可用的恢复手段。

---

## 8. 高级操作页（Advanced Operation）

高级操作页集中了风险更高的维护功能，例如：

- 与 AVB（Android Verified Boot）相关的操作；
- 某些 CVE v2 相关工具；
- 分区表导出、签名检查等。

由于实现细节会随版本演进，此处只给出高层说明，具体功能与警告请
以界面中的文字提示为准。总体原则：

- **所有高级操作默认视为危险操作**；
- 使用前务必阅读界面上的红色警告和说明；
- 尽量先在可恢复的测试设备上验证流程；
- 对生产/主力设备，请务必提前做好多份备份。

示例场景（仅示意）：

- 禁用或调整 AVB 相关参数，以便刷入非官方系统；
- 导出/修改分区表 XML 后在高级页面中重新导入。

> 再次强调：高级操作会绕过很多保护机制，一旦操作失误
> 设备极有可能陷入无法启动甚至无法再次识别的状态。

---

## 9. 高级设置页（Advanced Settings）

高级设置页主要用于配置全局行为：

- 块大小模式：选择自动、手动等；
- 某些与日志、调试相关的开关；
- 其他实验性功能开关等。

一般建议保持默认设置，仅在遇到特殊问题时，按照文档的说明
进行调整并记录更改前后的值，便于出现异常时回退。

---

## 10. 调试设置页（Debug Options）

调试设置页提供了一些用于问题排查和开发测试的选项，例如：

- 打开更详细的日志输出；
- 启用某些兼容性模式或实验性实现路径；
- 在某些操作上保留更多中间信息等。

这些选项可能显著改变性能或行为，建议只在需要时临时打开，
问题确认后再关闭。

---

## 11. 日志与关于

### 11.1 日志页（Log）

日志页会实时显示工具运行过程中的关键信息，包括：

- 连接与握手状态；
- 读取 / 写入进度与错误；
- PAC 解析和刷写阶段信息；
- 高级操作执行情况等。

遇到错误时，首先查看日志页中的最后几条信息，往往能提供
初步的错误原因（如权限问题、设备断开、镜像文件不匹配等）。

### 11.2 关于页（About）

关于页展示当前版本号和简要版权信息，并从
[docs/VERSION_LOG.md](docs/VERSION_LOG.md) 中读取更新记录摘要。

查看完整的版本历史和改动详情，请直接打开该文件。

## 12. 命令行模式简介

### 12.1 启动方式

在支持命令行的环境中（Linux / macOS / Termux 等），可以使用
以下命令以纯命令行方式启动：

```bash
sfd_tool --no-gui
```

查看所有可用命令与参数：

```bash
sfd_tool --help
```

在支持 man 手册的系统上，还可以使用：

```bash
man sfd-tool
```

### 12.2 常见命令示例

> 注意：下列仅为示例，实际可用命令以 `--help`/`man` 输出为准。

- 查看分区表：
  - 命令形式类似于 `part_table`/`cptable` 等；
- 读取指定区域：
  - 通过类似 `read_spec` 的命令，指定起始地址和长度，将数据读入指定文件；
- 在地址处执行代码：
  - 通过 `exec_addr` / `exec` 等命令，在给定地址执行预先写入的代码。

命令行模式通常不会给出过多保护提示，请在完全理解命令含义的
前提下使用。

> 建议在使用命令行模式前，先通过 GUI 熟悉基本流程和风险，
> 然后再把 GUI 里的操作映射到对应的 CLI 命令上。

## 13. 配置与语言

工具会在每个用户自己的目录下保存 JSON 配置文件，记录：

- UI 语言（`ui_language`）；
- 最近使用路径；
- 某些高级/调试开关状态等。

典型的 `ui_language` 取值：

- `auto`：跟随系统语言；
- `zh_CN`：简体中文；
- `en_US`：英文；
- 其他语言（若将来增加）会在文档中说明。

旧版本可能会在程序所在目录保存配置文件，新版在首次运行时
会尝试迁移到 per-user 目录。更多细节请参考：

- [docs/cmake.md](docs/cmake.md) 中关于配置与多语言的章节；
- `core/config_service` 相关实现（面向开发者）。

## 14. 常见问题与建议

- **设备无法识别或连接失败**
  - 检查数据线和 USB 接口；
  - 在 Windows 下检查驱动（LibUSB / SPRD）是否正确安装；
  - 在 Linux 下确认权限（root、udev 规则等）。

- **操作中途报错或进度停滞**
  - 在日志页查看最后几条记录；
  - 检查是否使用了不合适的块大小模式或手动参数；
  - 如为批量还原操作，确认镜像是否与设备型号完全匹配。

- **设备变砖 / 无法启动**
  - 尝试使用之前备份的分区镜像恢复；
  - 若完全无法识别设备，可能需要硬件方式（短接等）再次进入刷机模式，风险较高且超出本工具文档范围。

## 15. 总结

SFD Tool 提供了丰富而强大的分区操作与刷机功能，
同时也意味着一旦操作不当有较大风险。

**建议使用顺序：**

1. 首先通过 GUI 熟悉基本功能，在安全范围内进行备份和简单还原；
2. 逐步尝试 PAC 刷机和少量高级操作（务必先备份关键分区）；
3. 在充分理解风险后，才考虑使用命令行和更底层的功能；
4. 勿在主力设备上盲目尝试未验证的操作，多在测试设备上演练。

如需了解内部架构或参与开发，请参考：

- [ARCHITECTURE.md](ARCHITECTURE.md)
- [docs/cmake.md](docs/cmake.md)
- `core/`、`pages/`、`tests/` 等目录中的源码与测试用例。
