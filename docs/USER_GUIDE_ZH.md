# sfd_tool 使用手册

> 文档导航： [项目总览（README_ZH）](../README_ZH.md) · [架构文档](../ARCHITECTURE.md) · [CMake 使用指南](cmake.md) · [版本发布流程](RELEASE_GUIDE_ZH.md) · [版本记录](VERSION_LOG.md)

## 1. 工具简介

sfd_tool 是面向展讯 / 紫光展锐（Spreadtrum / UNISOC）设备的刷机与维护工具，提供：

- 通过 USB 连接 BROM / FDL 引导程序；
- 读取 / 写入分区（备份与恢复）；
- 刷写 PAC 固件；
- 修改分区表、AVB 相关参数等高级操作；
- 图形界面（GTK3）与命令行两种使用方式；
- 多语言界面（默认中文，可切换英文）。

更详细的架构设计可参考 [ARCHITECTURE.md](../ARCHITECTURE.md)。

## 2. 安装与启动

### 2.1 获取可执行文件

- **普通用户**：建议直接从 GitHub Release 页面下载已构建好的安装包或压缩包，按照对应平台说明安装。
- **开发者 / 需要自行编译**：请参考：
  - [CMake 使用指南（中文）](cmake.md)
  - [CMake Guide (EN)](cmake_EN.md)

### 2.2 启动 GUI

构建或安装完成后，在可执行文件所在目录运行：

- Linux / macOS：

  ```bash
  # 在拥有 USB 访问权限的情况下
  ./sfd_tool

  # 需要提升权限时（保持环境变量以便正确加载语言）
  sudo -E ./sfd_tool
  ```

- Windows：

  双击 `sfd_tool.exe` 即可启动（如从压缩包解压或安装程序安装后）。

程序首次启动会在当前用户的配置目录创建 `sfd_tool_config.json`，并默认将界面语言设置为中文。配置文件路径及语言切换方式详见 [CMake 使用指南](cmake.md) 中的“UI language and per‑user configuration”一节。

### 2.3 以命令行模式启动

若只希望使用命令行模式（无图形界面），可以使用参数 `--no-gui`：

```bash
./sfd_tool --no-gui
```

在 Termux 环境下的建议用法已在 [README_ZH](../README_ZH.md) 中给出，可直接参考。

## 3. 图形界面概览

主界面由若干个标签页（Tab）组成，每个标签页对应一类操作：

- **Connect**：设备连接与基本信息显示；
- **Partition Operation**：分区列表、读取 / 写入 / 备份分区；
- **PAC Flash**：选择 PAC 固件进行整机刷写；
- **Manual**：手动输入地址、长度等参数进行低层读写或执行镜像；
- **Advanced Operation**：高级 / 风险操作（如 AVB 相关处理等）；
- **Advanced Settings**：全局设置（语言、日志、超时等）；
- **Debug Options**：调试与实验性开关；
- **Log**：运行日志输出；
- **About**：版本信息、开源协议以及 [VERSION_LOG](VERSION_LOG.md) 中的变更摘要。

> 提示：部分页面命名会随界面语言变化，但位置与功能保持一致。

### 3.1 Connect 标签页

典型用途：

1. 连接 USB 设备并建立与 BROM / FDL 的通信；
2. 显示已连接设备的一些基本信息；
3. 作为进行后续读写、刷机操作前的起点。

常见流程：

- 确认设备通过 USB 连接到电脑，部分设备需要按特定按键组合进入刷机模式；
- 在 Connect 页面中点击“连接”或类似按钮；
- 如长时间无响应，可在 Log 页面查看详细日志，或检查系统权限（Linux 下是否使用 `sudo -E`、是否配置了 udev 规则等）。

### 3.2 Partition Operation 标签页

用于对单个或多个分区进行备份与恢复，典型操作包括：

- 读取指定分区到本地文件（如备份 `nv` / `modem` 等关键分区）；
- 将本地镜像写回分区；
- 查看分区表信息（起始地址、大小、类型等）。

操作建议：

1. 在 Connect 页面确保设备已成功连接；
2. 切换到 Partition Operation 页面；
3. 在分区列表中选择目标分区；
4. 选择“读取（Read）”或“写入（Write）”，并指定输出 / 输入文件路径；
5. 执行操作并观察进度与日志输出。

**强烈建议**：在进行任何写入操作前，对关键分区做一次完整备份。

### 3.3 PAC Flash 标签页

用于刷写官方或第三方 PAC 固件包。典型流程：

1. 在 Connect 页面连接设备；
2. 切换到 PAC Flash 页面；
3. 选择 PAC 文件；
4. 根据需要勾选 / 取消勾选某些分区或选项（如只刷系统分区等）；
5. 点击开始刷写并等待完成。

刷写 PAC 存在较高风险，请确保：

- PAC 文件来源可靠、版本与设备型号匹配；
- 刷写过程中不要断开 USB 或意外断电；
- 如刷机失败，优先从日志中查找具体错误原因。

### 3.4 Manual 标签页

Manual 页面提供更接近底层的手动操作入口，例如：

- 指定起始地址与长度，直接读取原始数据到文件；
- 指定镜像文件与执行地址，手动执行某段代码（FDL、引导代码等）；
- 配合命令行参数使用 `exec_addr` / `exec` 等能力。

该页面适合高级用户进行实验性或定制化操作，误用可能导致设备无法启动，请谨慎操作。

### 3.5 Advanced Operation / Advanced Settings / Debug / Log / About

- **Advanced Operation**：集中了一些高级功能（例如禁用 AVB、校验或修补某些安全属性等）。这些功能通常会修改设备安全相关配置，若不完全理解含义，请勿随意尝试。
- **Advanced Settings**：包括日志保存位置、语言、超时时间等全局设置。语言设置会写入 per-user 配置文件。
- **Debug Options**：开启或关闭某些调试开关，用于排查问题或配合开发版本使用。
- **Log**：实时显示操作日志，可用于观察刷机、读写过程以及错误信息。
- **About**：展示当前版本号、版权信息与简要的版本记录；详细变更历史可参见 [VERSION_LOG.md](VERSION_LOG.md)。

## 4. 常见操作示例

### 4.1 备份关键分区

1. 启动 GUI 并连接设备（Connect 页面）。
2. 切换到 Partition Operation 页面。
3. 在分区列表中选择需要备份的分区（如 `nv`、`modem`、`system` 等）。
4. 选择“读取（Read）”或类似按钮，指定输出文件路径。
5. 等待操作完成，并确认日志中无错误信息。

建议将备份文件妥善保存在安全位置，以便日后恢复。

### 4.2 使用 PAC 固件刷机

1. 备份关键分区（见上一小节）。
2. 在 Connect 页面确保设备已成功连接。
3. 切换到 PAC Flash 页面并选择对应机型的 PAC 文件。
4. 根据需要调整刷写选项（全量刷写 / 只刷系统 / 不覆盖某些分区等）。
5. 点击“开始”并耐心等待过程结束。
6. 操作完成后，设备通常会自动重启；如未重启，可手动重启并检查系统是否正常启动。

### 4.3 只使用命令行模式

在支持命令行的环境中（Linux / macOS / Termux 等），可以使用 `--no-gui` 参数启动：

```bash
./sfd_tool --no-gui
```

在 Termux 中，USB 权限的获取与传递方式请参考 [README_ZH](../README_ZH.md) 中的示例命令。

## 5. 命令行常用命令说明

在命令行模式下，sfd_tool 支持一系列子命令。**本节只列出在 README 中重点提到、且日常使用较多的部分命令**，用于帮助快速上手。

> 完整的命令和参数参考：
> - 运行 `./sfd_tool --help` 查看控制台内置帮助；
> - 在安装了 manpage 的 Linux 环境中，可以使用 `man sfd-tool`；
> - 其内容与源码中的 `main_console.cpp::print_help()` 保持一致，以代码实现为准。

- `part_table [文件地址]`
  等同于 `partition_list` 命令，将分区表导出到指定文件。

- `exec_addr [镜像文件] [地址]`
  需要同时提供镜像文件路径与执行地址，用于在指定地址执行某段代码。

- `exec <地址>`
  执行指定地址处的代码。执行 FDL1 时需要提供 FDL1 的地址。

- `read_spec [分区名] [偏移] [大小] [文件名]`
  等同于 `read_part` 命令，`read_part` 也可简写为 `r`。用于读取某分区指定偏移和长度的数据到文件。

- `--no-fdl`
  新增参数，用于在特定模式下（Sprd4 模式）免 FDL 刷机。

- `cptable`
  新增命令，以兼容性方法获取分区表（仅 FDL2 模式）。

- `--no-gui`
  以纯命令行方式启动工具。

> 提示：命令行模式通常不会给出太多“保护提示”，请在完全理解命令含义的前提下使用，误操作可能导致设备变砖。

## 6. 安全与权限提示

- 绝大多数操作都依赖对 USB 设备的低层访问权限，部分系统必须在 root / 管理员权限下运行；
- 请尽量避免在不熟悉的设备上进行高危操作（如修改分区表、禁用安全校验等）；
- 切勿在生产环境或包含敏感数据的设备上随意实验；
- 本工具在 GPL v3 许可证下发布，作者与贡献者不对使用过程中造成的任何后果负责。
